## 前言

本報告以五種經典排序演算法為研究對象：氣泡排序、選擇排序、插入排序、合併排序與快速排序。目標為說明各排序法的基本原理與操作方式，分析它們的時間與空間複雜度，並透過程式模擬比較不同資料規模下的排序效率。

## 1. 排序法原理

### 1.1 氣泡排序法 (Bubble Sort)
氣泡排序透過重複掃描陣列，逐一比較相鄰元素。若前者大於後者，則交換位置。每一輪結束後，最大的元素會像氣泡般「浮」到末端。

### 1.2 選擇排序法 (Selection Sort)
選擇排序將陣列分成已排序與未排序兩部分。每一輪在未排序區間中找出最小值，並將該最小值交換到目前未排序區間的第一個位置。

### 1.3 插入排序法 (Insertion Sort)
插入排序把陣列分成已排序區段與未排序區段。每次取出未排序區段的第一個元素，從已排序區段右端開始比較，並插入到正確位置。

### 1.4 合併排序法 (Merge Sort)
合併排序屬於分治法。它先將陣列不斷對半切分，直到每個子陣列只有一個元素，然後再兩兩合併，合併時保持結果有序。

### 1.5 快速排序法 (Quick Sort)
快速排序先選擇一個基準值（pivot），將陣列重新排列成左側小於基準、右側大於基準，接著遞迴對左右子陣列執行相同步驟。

## 2. 複雜度分析

| 排序法 | 平均時間複雜度 | 最好時間複雜度 | 最壞時間複雜度 | 空間複雜度 | 穩定性 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| 氣泡排序 | $O(n^2)$ | $O(n)$ | $O(n^2)$ | $O(1)$ | 穩定 |
| 選擇排序 | $O(n^2)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 不穩定 |
| 插入排序 | $O(n^2)$ | $O(n)$ | $O(n^2)$ | $O(1)$ | 穩定 |
| 合併排序 | $O(n \log n)$ | $O(n \log n)$ | $O(n \log n)$ | $O(n)$ | 穩定 |
| 快速排序 | $O(n \log n)$ | $O(n \log n)$ | $O(n^2)$ | $O(\log n)$ | 不穩定 |


- 氣泡排序、選擇排序、插入排序屬於 $O(n^2)$ 之簡單排序，適合小規模資料。
- 合併排序與快速排序屬於 $O(n \log n)$ 之高效排序，適合較大資料量。
- 空間複雜度方面，合併排序需要額外暫存空間，而其他三種原地排序只需常數空間。

## 3. 數據模擬

本專案透過五個排序模擬頁面，分別展示氣泡排序、選擇排序、插入排序、合併排序與快速排序的執行過程。每個模擬頁面皆可：
- 自動生成或輸入測試資料
- 演示排序過程中的每一步比較與交換
- 顯示排序完成後的統計數據，如比較次數、交換次數、分割次數、總步數與耗時

模擬過程不僅可視化排序邏輯，也有助於觀察不同演算法在相同資料上操作行為的差異。特別是在資料量增加時，$O(n^2)$ 排序法的步驟數與交換次數明顯成長，而 $O(n \log n)$ 排序法的分割與比較次數則呈較緩和上升。

實驗中使用的數據為隨機整數序列，並針對三種資料規模進行多次測量，藉此取得平均效能，減少單次模擬的隨機波動。

### 3.1 氣泡排序模擬

氣泡排序在 n = 10 時平均比較次數為 45 次，交換次數約 20.55 次；隨著 n 增加到 20 與 50，平均比較次數分別上升至 190 與 1225 次，交換次數則分別增至 91.35 與 641.3 次。這顯示氣泡排序在每一輪中都會重複掃描陣列，相鄰元素比較頻繁，因此適合資料量極小的情況。

### 3.2 選擇排序模擬

選擇排序在 n = 10 時平均比較次數為 45 次，交換次數僅 7.35 次；n = 20 時比較次數為 190 次、交換次數 16.25 次；n = 50 時比較次數達 1225 次、交換次數 45.55 次。選擇排序每一輪只執行一次交換，交換次數較少，但比較次數仍隨 n 的平方成長，顯示它對資料量增長較不友善。

### 3.3 插入排序模擬

插入排序在 n = 10 時平均比較次數為 27.75 次、插入次數 9 次；當 n = 20 與 50 時，平均比較次數分別為 108.3 與 687 次，插入次數分別為 19 與 49 次。插入排序在小規模或接近有序資料時表現良好，但隨著資料不斷變大，元素插入成本仍會顯著上升。

### 3.4 合併排序模擬

合併排序在 n = 10 時平均比較次數為 22.4 次，當 n = 20 與 50 時分別為 63.05 與 222.7 次。由於合併排序採用分治法，遞迴切分與合併步驟使比較次數維持在 $O(n \log n)$ 水平，雖然需要額外空間，但比較效率比簡單排序更穩定。

### 3.5 快速排序模擬

快速排序在 n = 10 時平均比較次數 24.1 次、分割次數 6.6 次；當 n = 20 與 50 時，平均比較次數分別為 70.9 與 259 次，分割次數分別為 13 與 32.95 次。快速排序的分割次數反映了遞迴深度，在平均情況下仍較合併排序更省空間，且執行時間隨資料規模增加時優勢明顯。

### 5.2 效能觀察
- **比較次數分析**：隨著 n 增加，三種 $O(n^2)$ 排序的比較次數急速上升，尤其是氣泡排序與插入排序。合併排序與快速排序的比較次數增長較緩，符合 $O(n \log n)$ 的趨勢。
- **交換/插入次數分析**：選擇排序的交換次數最少，因為每輪最多只交換一次。氣泡排序與插入排序的交換/插入次數隨 n 增加而顯著上升。
- **分割次數分析**：快速排序的分割次數反映其遞歸深度，平均情況下效率較高。
- **執行時間分析**：對於小規模資料 (n ≤ 50)，所有排序演算法的執行時間都很短 (≤ 0.08 ms)，難以區分差異。但隨著資料規模增加，$O(n \log n)$ 演算法的時間優勢將更明顯。
- **綜合比較**：插入排序在 n = 10 時比氣泡排序更優，但隨著 n 增加仍舊落後於合併排序與快速排序。選擇排序在交換次數上最有效率，但比較次數與氣泡排序相同。

### 5.3 n 增加時的變化
- **比較次數變化**：$n = 10$ 到 $n = 20$：氣泡、選擇與插入排序的比較次數大約增加 4 倍，顯示 $O(n^2)$ 的增長特性。$n = 20$ 到 $n = 50$：同樣三種排序的比較次數顯著增加，快速排序與合併排序的成長幅度較小。
- **交換/插入次數變化**：氣泡排序與插入排序的交換/插入次數隨 n 增加而急劇上升，選擇排序的交換次數增加較緩慢。
- **分割次數變化**：快速排序的分割次數隨 n 增加而增加，但增長率低於比較次數。
- **執行時間變化**：$n = 10$ 到 $n = 20$：執行時間略有增加，但仍在 0.01 ms 左右。$n = 20$ 到 $n = 50$：$O(n^2)$ 演算法的時間增加較明顯，氣泡排序達到 0.08 ms。
- **結論**：資料規模越大，$O(n \log n)$ 排序在比較次數和執行時間上的相對優勢越明顯。

## 6. 心得與結論
這次實驗透過理論與模擬實驗觀察，驗證了不同排序法再耗費時間和複雜度的差異，對於較小規模的資料，氣泡和選擇排序的實作較其他簡單，但在規模變大時，交換次數明顯高於合併和快速排序，對於大規模資料，合併排序雖然需要額外空間，但能穩定維持較低比較次數；快速排序在平均情況下也具備最佳效能，且對於資料規模擴大時具有更明顯的時間優勢，而插入排序在較有序的資料上效率較好。

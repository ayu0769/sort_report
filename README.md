## 1. 五個排序法介紹

### 氣泡排序法 (Bubble Sort)
透過重複走訪資料陣列，逐一比較相鄰元素。若前者比後者大，則交換位置。每一輪走訪後，最大的元素會「浮」到陣列末端。

### 選擇排序法 (Selection Sort)
每一輪從未排序區段中選出最小值，與未排序區段的第一個元素交換。隨著輪次增加，已排序區段逐漸擴大。

### 插入排序法 (Insertion Sort)
將資料分成已排序區段與未排序區段。每次取出未排序區段的第一個元素，從已排序區段右側開始比較並插入到正確位置。

### 合併排序法 (Merge Sort)
採用分治法。先將陣列不斷切分成小區段，直到每個區段只剩一個元素，然後再兩兩合併，合併時保持結果有序。

### 快速排序法 (Quick Sort)
先選擇一個基準元素（pivot），將陣列分成小於基準值與大於基準值的兩部分，然後遞迴地對這兩部分繼續進行快速排序。

## 2. 複雜度分析

| 排序法 | 平均時間複雜度 | 最好時間複雜度 | 最壞時間複雜度 | 空間複雜度 | 穩定性 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **氣泡排序** | $O(n^2)$ | $O(n)$ | $O(n^2)$ | $O(1)$ | 穩定 |
| **選擇排序** | $O(n^2)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 不穩定 |
| **插入排序** | $O(n^2)$ | $O(n)$ | $O(n^2)$ | $O(1)$ | 穩定 |
| **合併排序** | $O(n \log n)$ | $O(n \log n)$ | $O(n \log n)$ | $O(n)$ | 穩定 |
| **快速排序** | $O(n \log n)$ | $O(n \log n)$ | $O(n^2)$ | $O(\log n)$ | 不穩定 |

## 4. 模擬實作內容

### 氣泡排序法
`bubblesort.html` 中採用逐步比較相鄰元素的方式，若前者大於後者則交換。每輪結束後，最大元素會標記為已排序並固定在末端。

### 選擇排序法
`selectionsort.html` 中以目前索引為起點，搜尋剩餘元素的最小值。若找到更小的值，則與目前位置交換，並向右移動已排序邊界。

### 插入排序法
`insertionsort.html` 中從第二個元素開始，向前比較並依序交換，直到該元素插入到已排序區段的正確位置。

### 合併排序法
`mergesort.html` 中使用遞迴分割陣列，再將左右子序列以比較方式合併成有序結果。合併過程中會記錄每次比較與合併完成的端點區間。

### 快速排序法
`quicksort.html` 中選擇最後一個元素作為基準值，將陣列分成小於與大於基準值的兩部分，並在遞迴中持續分割與排序左、右子區段。

## 5. 結果呈現與比較

- 中小型資料集下，快速排序與合併排序的比較次數顯著低於氣泡、選擇與插入排序。
- 氣泡排序與選擇排序的比較次數相同，但選擇排序的交換次數通常較少，因為每輪最多交換一次。
- 插入排序在接近有序資料時效率較高；本次隨機資料測試中，其比較次數仍少於氣泡與選擇排序。
- 合併排序穩定且比較次數成長趨近 $O(n \log n)$，但需要額外儲存空間進行合併。
- 快速排序平均性能良好，但在最壞情況下仍可能退化為 $O(n^2)$。
